# Stage 47 — UX-аудит игрового стола

## Текущие проблемы

### Игровой стол

- Игроки не выглядят так, будто сидят за столом.
- Позиции игроков плохо читаются при разном количестве игроков.
- На mobile элементы стола могут накладываться друг на друга.
- Центр стола перегружен текстом.
- Собранные квартеты показываются слишком технически.
- Последние события занимают место внутри стола и мешают композиции.
- Игровой стол выглядит как обычная панель, а не как центральная игровая зона.

### Action-блок текущего хода

- Блок запроса карты отделён от контекста стола.
- Состояние “твой ход” дублируется в нескольких местах.
- Кнопка запроса карты выглядит как обычная форма, а не как основное действие хода.

### Request flow

- Модальное окно выглядит как техническая форма.
- Выбор игрока занимает слишком много места при одном сопернике.
- Карты меняют визуальный вес в зависимости от количества доступных карт.
- При малом количестве карт сетка выглядит пустой и несбалансированной.
- Выбранный игрок и выбранная карта должны быть заметнее.
- На mobile модалка требует отдельной адаптации.

## Что нужно получить после Stage 47

- Игровой стол с понятным seating layout для 2, 3 и 4 игроков.
- Игроки визуально расположены вокруг стола.
- Центр стола показывает только важное состояние партии.
- Последние события не ломают композицию стола.
- Action-блок текущего хода связан со столом.
- Request flow выглядит как игровой выбор, а не как обычная форма.
- Размер карт в request flow стабильный.
- Mobile layout читаемый и без наложений.

## Техническое решение

- Остаёмся на HTML/CSS.
- Canvas пока не используем.
- Seating layout делаем через CSS-классы по количеству игроков.
- Request flow улучшаем через фиксированную сетку карточек и отдельные selected-состояния.
