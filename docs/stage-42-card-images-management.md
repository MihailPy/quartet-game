# Stage 42 — Card Images Management

## Цель

Добавить поддержку изображений карт: хранение ссылки на изображение, загрузка
изображения через UI и отображение изображения в игровом интерфейсе.

## Модель карты

У каждой карты появляется необязательное поле:

```json
{
  "image_url": "string | null"
}
```

## Правила

* image_url может быть пустым.
* Если изображения нет, показывается fallback.
* Изображение не должно быть обязательным для старых карт.
* Загрузка изображения должна быть доступна через UI управления картой.
* В gameplay используется только уже сохранённое изображение.

## Где отображается изображение

* Card preview modal
* Player hand
* Request card flow
* В будущем: deck editor / card management UI

## Форматы

Поддерживаемые форматы:

* .jpg
* .jpeg
* .png
* .webp

## Ограничения

Рекомендуемый лимит файла:

* до 5 MB

Размеры изображений

Базовый стандарт:

* aspect ratio: 5:7
* recommended source: 1500×2100 px
* minimum acceptable: 750×1050 px

## Fallback

Если у карты нет image_url, UI показывает текущий placeholder.
