# Stage 48. Gameplay UX Foundation

## Цель

Создан фундамент нового игрового интерфейса:
единая модель состояния, игровой стол, игроки и центральный статус.

## Итоговая структура компонентов

GameplayScreen

├── GameplayStatusBar
├── GameplayTable
│   └── GameplayPlayerCard
├── GameplayCentralStatus
├── ActionPanel
└── PlayerHandPanel

## Источники состояния

Основной источник:

PublicGameState

Используется для:

- текущего игрока;
- текущего хода;
- количества карт;
- квартетов.

GameplayViewModel формируется через:

buildGameplayUIViewModel

## Поддержка игроков

Поддерживается:

- 2 игрока
- 3 игрока
- 4 игрока
- 5 игроков
- 6 игроков

2-4 игрока:
классическая раскладка вокруг стола.

5-6 игроков:
компактная сетка.

## Ограничения

Не реализовано:

- новая рука карт;
- inline request flow;
- мобильная оптимизация игровой зоны;
- анимации;
- звук.

## Перенос на Stage 49

Следующий этап:

- новая визуальная рука игрока;
- выбор карты;
- новый request flow;
- улучшение мобильного UX.
