# Gameplay Layout

## Goal

Gameplay screen should feel like a card table, not like a set of admin panels.

## Main Zones

### Top zone

Contains:

- player avatar/menu;
- latest 1–2 game events;
- optional connection/status indicators.

### Center table zone

Contains:

- player seats around the table;
- card backs for opponent card counts;
- completed quartets;
- request feedback area;
- final game result.

### Bottom hand zone

Contains:

- current player's hand;
- horizontal scroll;
- slight card overlap;
- card preview on click/tap.

## During Game

The main focus should be:

1. current turn;
2. visible players;
3. player hand;
4. request action.

## Finished Game

After game finish:

- show final scoreboard;
- show winners;
- keep event log available;
- hide active turn/request controls.

## Future Enhancements

- card request animations;
- card transfer animations;
- sound feedback;
- vibration feedback on mobile;
- card image previews;
- collectible card metadata.
